How GPU telemetry is detected
The dashboard reads local OS and vendor telemetry. If those tools are missing, blocked, or cannot see the device, there may be nothing to display.

Where the data comes from

NVIDIAUses the NVIDIA driver tools, usually nvidia-smi. On Jetson devices, telemetry comes from Jetson tools such as tegrastats.
IntelUses Intel GPU driver telemetry. On Linux this commonly comes through intel_gpu_top; on Windows it comes from GPU performance counters.
AMDUses AMD telemetry tools when available, such as rocm-smi on supported Linux systems.
WindowsUses Windows GPU performance counters and driver metadata, with vendor tools used when they provide better coverage.

Things to check

Confirm the GPU or NPU appears in the operating system device list.
Install or update the vendor driver, then restart the dashboard process.
Make sure the dashboard user can run the telemetry command, such as nvidia-smi, intel_gpu_top, rocm-smi, or tegrastats.
If the device is remote, containerized, or running under restricted permissions, confirm the hardware and driver tools are visible inside that environment.

Why this can happen

Some machines genuinely do not have a GPU or NPU exposed to the OS. Other times the device exists, but the driver utility is missing, requires elevated permissions, or reports a format the dashboard does not yet understand.
